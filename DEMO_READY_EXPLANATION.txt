DEMO-READY PROJECT EXPLANATION
ROS2 Path Planning and Maze Solving

1) Big Picture Architecture
The pipeline is:
1. Gazebo simulation publishes camera feeds and odometry.
2. Solver node receives top camera and bot camera.
3. Localization extracts robot position and occupancy map.
4. Mapping converts occupancy map to graph.
5. Path planning finds shortest path (Dijkstra or A*).
6. Motion planning converts path to velocity commands.
7. Robot moves via cmd_vel in Gazebo.

Why this matters:
- This separation into modules makes the system explainable and debuggable.
- If movement fails, you can isolate whether the issue is perception, graph building, planning, or control.
- It also makes comparison experiments easy, such as swapping Dijkstra and A* without changing localization.

5) Path Planning: Dijkstra vs A*
Path planning module:
- path_planning_ws/src/maze_bot/maze_bot/bot_pathplanning.py:27

Dijkstra:
- Optimal with non-negative weights.
- Tends to explore more nodes.
- Node-visit counter:
  path_planning_ws/src/maze_bot/maze_bot/bot_pathplanning.py:337

A*:
- Uses cost-so-far plus heuristic.
- Heuristic is Euclidean distance:
  path_planning_ws/src/maze_bot/maze_bot/bot_pathplanning.py:438
- Usually explores fewer nodes.
- Node-visit counter:
  path_planning_ws/src/maze_bot/maze_bot/bot_pathplanning.py:436

Why both are useful:
- Dijkstra is the correctness baseline and helps validate graph quality.
- A* is typically faster and better for scaling to larger mazes.
- Keeping both allows fair algorithm comparison in the same environment.

14) Common Q&A for Presentation
Q1) Why A* if Dijkstra already works?
A1) Dijkstra is a reliable optimal baseline; A* is usually faster due heuristic guidance.

Q2) Is the maze map pre-programmed?
A2) The graph used by planner is generated online from camera occupancy processing.

Q3) Why small startup movement?
A3) Short heading calibration helps align control behavior before full path following.

Q4) What if first frames are noisy?
A4) Localization includes guards to skip processing until valid contours/background are available.

Q5) Why did Gazebo duplicate-node errors happen before?
A5) Camera plugin name collisions; fixed by unique plugin names.

Why this matters:
- Preparing answers in advance increases confidence and technical clarity.
- It helps you handle faculty questions consistently under time pressure.
- Good Q&A performance signals system understanding, not just implementation.
